## Recommended Order

1. Preserve the rescue bundle itself.
   Copy or archive `artifacts/worktree-rescue/` before touching the linked worktrees.

2. Decide whether the backup branch should remain as a branch.
   Keep `backup/pre-merge-20260504-0003` until you are satisfied the patch is no longer needed.

3. Rehydrate only the patterns you care about.
   Apply one patch at a time onto a fresh branch or fresh worktree, not onto `main`.

4. Deduplicate the portal patterns.
   `portal-021-core-pattern` and `portal-021-css-pattern` overlap heavily.
   Start with the CSS variant if you want the fuller portal change set.

5. Only after rescue, clean the noisy worktrees.
   Most of the remaining dirtiness is repeated generated churn, screenshots, `node_modules`, caches, or submodule drift.

6. Clean local bytecode churn on `main` last.
   The remaining `scraper/__pycache__/` changes in the main checkout are generated files, not source work.

## Safe Recovery Workflow

Use a fresh branch and apply a single patch at a time.

Example:

```bash
git switch -c rescue/portal-021-css
git apply --index artifacts/worktree-rescue/portal-021-css-pattern.patch
git status
```

If a patch does not apply cleanly, use a throwaway worktree or branch and inspect the rejects before retrying.

## Cleanup After Rescue

After you confirm the needed patches are recovered:

1. remove or reset the dirty linked worktrees
2. keep or delete the backup branch depending on whether you still want the raw branch reference
3. remove generated bytecode churn from `main`
4. optionally keep `artifacts/worktree-rescue/` as an audit trail until all rescued work is committed elsewhere
